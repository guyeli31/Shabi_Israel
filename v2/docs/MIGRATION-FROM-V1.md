| 2026-06-30 | (staged) | **iOS search-sheet ‚Äî sidesteps Safari's focus-zoom on sub-16px inputs (per-platform: iOS overlay, Android inline-enlarge).** NEW `js/render/searchOverlay.js` + `css/search-overlay.css`. On **iOS only** (detected once ‚Üí `html.is-ios`; `?searchoverlay=force`/`=off` override for desktop testing), a capture-phase `mousedown` `preventDefault` blocks the inline `.app-search-input` from focusing (so iOS never zooms ‚Äî classic keep-focus trick; mousedown not pointer/touch so the click survives), and a capture-phase `click` (`stopPropagation`, so the field's own focus/click handlers never fire) opens a `position:fixed` sheet pinned to `visualViewport.offsetTop` whose input is **`font-size:16px`** (at threshold ‚Üí no zoom). Smart-search results render below; picking either navigates (sidebar) or writes the value back into the source field + dispatches `input` so its existing handler runs in place. The page never zooms and never scrolls, so closing restores the exact position with the field filled.
